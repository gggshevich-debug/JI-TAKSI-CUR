 Тексты Web Push

            

              Шаблоны для событий приложения. Плейсхолдеры в фигурных скобках, например
              {trip_id}, {price},
              {driver_label}, {taxi_name},
              {start_address},
              {addressFrom} / {address_from},
              {addressTo} / {address_to} / {end_address},
              {distanceKm} / {distance_km} (км, два знака после запятой);
              выводы: {withdrawal_id}, {amount}, {card_last4}, {reason}.
              Поле «Подзаголовок» поддерживается не везде; на iOS у Web Push система часто добавляет строку вида «from …» — это ограничение платформы, а не текста шаблона.
              После сохранения новые тексты используются при следующей отправке.
            

            
            

          

          

            
 Тестовая отправка

            

              Отправляет push на все сохранённые подписки указанного пользователя (как в приложении после «Разрешить уведомления»).
              Нужны переменные VAPID_* на сервере.
            

            
            
            
            
            
            
            
            

          

        

      


      

        

          

            
 Тарифы проекта (сервер)

            

              Итоговая цена клиента: база + сумма по ступеням ₼/км (JSON) + минуты × тариф;
              затем не ниже минимума; для длинных поездок — потолок базы по ₼/км, затем множители (для длинных ослаблены), затем финальный потолок (км × ставка × коэффициент).
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              
            

            

              Волны, SOLO, приоритет и анти-спам офферов
              

                Волны: доли кандидатов по волнам, таймаут волны, минимум водителей в волне 1, интервал между офферами одному водителю (глобально) и жёсткий cooldown повтора по тому же заказу (Redis).
                Цена: surge по tier и по времени с первого оффера; клиентский мягкий буст из приложения.
                SOLO: пороги acceptance/рейтинга для одиночного первого оффера.
                Скоринг: нормализация метрик, вес расстояния (в т.ч. авто от плотности), «рядом хороший» ограничивает выбор дальних, бонус стабильности EMA accept, штрафы отказов по типам (Redis × масштаб).
